Introdução

Sobre este Documento

Este documento irá descrever a estratégia, visão e direção do projeto Xubuntu, assim como servirá como guia e uma referência para o time do Xubuntu.

Esta versão revisada tem sido escrita por Pasi Lallinaho,com a ajuda de Elizabeth Krumbach e Simon Steinbeiß, junto com várias pessoas da comunidade. Esta versão é baseada no antigo Xubuntu Strategy Document, escrito por Cody Somerville et al.

Nós gostariamos de agradecer a Cody Somerville, Jono Bacon, Eero Tamminen, Nico Veenkamp e Lionel Le Folgoc, pelas suas contribuições a este documento. Alguns conteúdos, ideias e inspiração foram derivados da documentação existente do Ubuntu.

Declaração de Missão

O Xubuntu é um sistema operacional baseado no Ubuntu, desenvolvido pela comunidade, que combina elegância e facilidade de uso. Ele oferece um desktop leve, estável e personalizável com fluxos de trabalho conservadores. O Xubuntu oferece um produto polido e unificado, pronto para usuários finais.

Para além dos aspectos técnicos, o time do Xubuntu se concentra nas comunidades de contribuintes e usuários.

Objetivo

O público-alvo do Xubuntu são usuários que estão interessados em ter um sistema operacional elegante, fácil de usar, polido e unificado. O Xubuntu é uma boa opção para aqueles que querem um desktop estável, personalizável e/ou relativamente leve. Finalmente, o Xubuntu é uma escolha atraente para usuários que preferem fluxos de trabalho conservadores em detrimento das inovações recentes.

O Xubuntu não objetiva atender usuários com conhecimentos ou habilidades específicas. Nós queremos que o Xubuntu seja uma opção viável para usuários que são novos no Linux ou na plataforma Ubuntu, mas que também seja atraente para usuários experientes. O Xubuntu não foca especificamente em novos usuários ou usuários migrando do Windows, mas por ser um sistema de fácil utilização, ele acaba se tornando uma boa alternativa para estes usuários também.

O Xubuntu não objetiva explicitamente atender usuários com máquinas baixo, modesto ou alto processamento mas, ao invés disso, ele tem como alvo todo o espectro. A capacidade de resposta e a rapidez do Xubuntu, dentre outras características positivas, podem ser apreciadas por todos os usuários, independente do hardware de suas máquinas.

Apesar do Xubuntu utilizar o Xfce, ele não é especificamente direcionado a entusiastas do Xfce ou projetos e softwares hospedados pelo projeto Xfce ou associados com o Xfce (oficialmente ou não), uma vez que não há garantias de inclusão destes softwares no Xubuntu.

Áreas de Foco

Esta parte do documento estratégico oferece uma estrutura transparente e um conjunto de prioridades para todo o desenvovimento do Xubuntu e suas tarefas, incluindo (mas não limitado a) processos de tomada de decisão, seleção de pacotes e resolução de conflitos técnicos.

As áreas de foco são apresentadas de forma abstrata e simples, mas também são concretas o suficiente para serem acionáveis. Como fundamento da visão do Xubuntu, as especificações no documento estratégico não devem ser confundidas com os objetivos e prioridades específicos de cada versão lançada, mas sim reconhecidos como os princípios que baseiam os objetivos específicos de cada versão do Xubuntu.

Foco 1: Usabilidade

Usabilidade é um dos objetivos mais importantes de um sistema operacional que é utilizado diariamente. Por isso, o Xubuntu deve ser fácil de usar e ter uma aparência que não atrapalhe a usabilidade. Este nível de aparência deverá ser observado em todas as interfaces de usuário, desde o boot até o desligamento.

O Xubuntu deve ser traduzido, para permitir que os usuários o utilizem em seus idiomas preferidos. Outro aspecto importante da usabilidade é a customização. Nós acreditamos que o Xfce oferece aos usuários a possibilidade de configurar seu sistema de maneira significativa; outros aplicativos devem conviver com este aspecto do Xfce de forma moderada.

Apesar da acessibilidade não estar entre as prioridades do Xubuntu, ela deve ser levada seriamente em consideração, desde que não exija esforços inviáveis para integrá-la, implementá-la ou mantê-la. Pelo menos as ferramentas de acessibilidade mais comuns devem vir na instalação padrão do Xubuntu.

Foco 2: Desempenho

A equipe do Xubuntu deve se esforçar para fazê-lo ser leve. Isto significa que cada lançamento do Xubuntu deve funcionar moderadamente bem em máquinas antigas, cuja fabricação foi poucos anos antes da data de lançamento da versão. Consequentemente, os lançamentos recentes do Xubuntu podem não suportar computadores muito antigos. Porém, estes lançamentos asseguram que a equipe do Xubuntu é capaz de trabalhar em outras melhorias e oferecer um sistema que é capaz de corresponder às expectativas de um sistema operacional moderno.

Os usuários que desejam o sistema mais leve possível devem ser direcionados para o CD "minimal", distribuições mais leves (tais como o Lubuntu) ou outras opções.

No início do ciclo de cada lançamento, os requisitos mínimos do sistema devem ser reavaliados para determinar se eles continuam sendo realistas.

Foco 3: Pronto para Uso

É muito importante, para nós, oferecer um produto polido e unificado que esteja pronto para usuários finais. Isso também é um prerrequisito para permitir que o Xubuntu seja um desktop útil, prático e eficaz. Sem a integração mencionada, o Xubuntu parecerá áspero e não polido, o que não o tornaria atraente para usuários finais.

A integração é acompanhada, também, por uma seleção de aplicativos e bibliotecas que trabalham bem entre eles, bem como pela aplicação de patches que asseguram um sistema livre de bugs.

Veja Seleção de pacotespara diretrizes sobre como determinar se um aplicativo se encaixa na pilha do Xubuntu.

Foco 4: Comunidade

A comunidade do Xubuntu é uma força importante na criação dele, que faz ele ser o mais perfeito possível. Para isto, é essencial que a coordenação entre os colaboradores funcione bem. A infraestrutura para permitir esta boa comunicação entre os colaboradores e usuários está descrita mais precisamente em Comunidade e Desenvolvimento.

Assim que possível e apropriado, a equipe do Xubuntu deve trabalhar junta a outras comunidades, incluindo outras equipes do Ubuntu e além. Se a cooperação em um dado momento não for possível, mas for benéfica para ambas as partes, a equipe do Xubuntu deve tentar alocar recursos para cooperação em um outro momento.

Comunidade

Conselho do Xubuntu

O Conselho do Xubuntu (mais tarde: conselho) é responsável por supervisionar a direção de desenvolvimento de Xubuntu e permitir que a comunidade trabalhe com Xubuntu o mais harmoniosamente possível. O conselho serve um termo de quatro liberações/dois anos de cada vez, terminando com uma liberação de LTS para permitir o planejamento a longo prazo.

A equipe do Launchpad do xubuntu-council é composta pelos membros do conselho. A equipe xubuntu-council é a administradora de todas as outras equipes Xubuntu Launchpad, dando ao conselho acesso total à administração para as equipes do Xubuntu.

Membro

O conselho é composto por 3 membros, que são escolhidos por um voto CIVS. Todos os membros em Equipe do Xubuntu e/ou membros diretos de quaisquer sub-equipes moderadas podem ser nomeadas, por si ou por qualquer outra pessoa na comunidade Xubuntu com o consentimento dos candidatos. O mesmo grupo de pessoas está qualificado para votar.

Se um membro do conselho ficar inativo por 6 meses, o membro será substituído por um novo membro do conselho pelo período restante, eleito por um novo voto CIVS.

O conselho decidirá sobre uma cadeira para todo o mandato do conselho. A cadeira atuará como o ponto de contato oficial para Xubuntu.

Operação do conselho

Espera-se que o conselho tome medidas ou responda a qualquer questão levantada dentro de 2 semanas. Se apropriado e justo, a primeira ação pode ser adiada.

Se em qualquer momento uma maioria da equipe Xubuntu considerar as ações do conselho prejudiciais para o projeto, o problema em questão deve ser levantada para o conselho para reconsideração. Se uma resolução não puder ser encontrada, o conselho deve consultar o Conselho da Comunidade do Ubuntu e outras equipes apropriadas da comunidade do Ubuntu. Da mesma forma, se o conselho não puder chegar a um consenso, mesmo depois de consultar a equipe Xubuntu, eles devem consultar o Conselho Comunitário do Ubuntu e outras equipes apropriadas da comunidade Ubuntu .

Equipe do Xubuntu

A equipe do Xubuntu é um grupo de indivíduos que são os principais responsáveis pelo desenvolvimento do Xubuntu.

A equipe Xubuntu é organizada no Launchpad como xubuntu-team. A equipe do Launchpad é moderada.

Os indivíduos que desejarem aderir à equipe Xubuntu são obrigados a cumprir os critérios descritos abaixo antes de aplicar ou eles serão automaticamente recusados. Líderes para subequipes podem ser nomeados nesta equipe. A equipe Xubuntu é moderada pelo conselho.

Caso um líder de equipe seja minoritário ao votar em uma questão que diga respeito à sua área de atuação, o líder da equipe deve receber outra audiência, após o que a equipe tiver tomado cuidado extra ao decidir sobre o assunto e pode querer considerar a votação novamente depois de mais discussão. Se necessário, a questão pode ser levantada para o conselho para posterior consideração.

Tornando-se um Membro e Permanecendo

Para serem aceitos pela equipe Xubuntu, os participantes devem demonstrar sua motivação e capacidade de contribuir para o Xubuntu. Isso é para garantir que qualquer membro da equipe Xubuntu tenha uma compreensão suficiente da comunidade Xubuntu e sua operação. As diferentes etapas que devem ser seguidas também indicam que o membro candidato é capaz de trabalhar dentro das diretrizes e, mais importante, com outras pessoas e comunidades. Para se tornar um membro, deve-se:

  • Comprometer contribuições significativas para uma das subequipes, após o que se pode ser aprovado para a subequipe para "provação" por um administrador de subequipe.

  • Demonstrar motivação para contribuir perpetuamente, após o qual pode ser aprovado para a equipe Xubuntu

Para continuar sendo um membro da equipe Xubuntu, o membro terá que ter o desejo de continuar a contribuir no futuro. Qualquer pessoa que não tenha contribuído mais do que um ciclo completo (6 meses) deve desativar-se da equipe, bem como todas as sub-equipes e reaplicar se/quando eles estiverem dispostos a voltar. Os administradores da equipe devem limpar as pessoas sem contribuições em mais de 6 meses das equipes ocasionalmente.

Usuários Xubuntu

A equipe do Launchpad xubuntu-users é o grupo formal de alto nível de pessoas que estão interessadas no Xubuntu. A equipe está aberta para qualquer pessoa se juntar.

Os membros do grupo oficial do Launchpad são encorajados a dar suporte ao usuário no IRC e na lista de discussão dos usuários do Xubuntu, bem como a espalhar a palavra sobre o Xubuntu em qualquer lugar que acharem conveniente, incluindo mas não limitado a blogs, mídias sociais e conferências.

Além de suporte ao usuário e advocacia, os membros são livres para organizar como eles gostam em plataformas de mídia social e muito mais. Se o grupo é considerado substancial e útil o suficiente, a equipe Xubuntu pode nomeá-lo oficial e ajudar com a manutenção. Todas as equipes oficiais e seus administradores estão listadas no wiki do Xubuntu na página Administradores do site.

Desenvolvimento

Coordenação

A direção do desenvolvimento do projeto é coordenada por:

  • O documento estratégico

  • Modelos e especificações específicos da versão

  • As reuniões da comunidade

  • As ferramentas de comunicação do desenvolvedor listadas abaixo

Comunicação

Para que o Xubuntu seja bem sucedido, seus membros e comunidade devem ter as ferramentas certas para permitir uma comunicação útil e, em última análise, ajudar a comunidade a crescer. Essas ferramentas de comunicação incluem as seguintes ferramentas "básicas":

  • Lista de discussão; Xubuntu-users para suporte comunitário e discussões de usuários e a lista Xubuntu-devel para discussão e coordenação de desenvolvimento

  • Canais de IRC; #xubuntu para suporte comunitário, #xubuntu-devel para discussão e coordenação de desenvolvimento e #xubuntu-offtopic para discussão geral com um humor mais descontraído

  • Launchpadpara organização da equipe e concessão de permissões e direitos de acesso através de membros da equipe

  • O site do Xubuntupara notícias,artigos de desenvolvimento e informações gerais sobre o Xubuntu

  • A área de desenvolvedor do Xubuntu para acompanhar o progresso do desenvolvimento e colaborar em questões de desenvolvimento e compartilhar informações relacionadas ao desenvolvimento na wiki

Além de se comunicarem uns com os outros e com a comunidade Xubuntu, a equipe Xubuntu deve se comunicar continuamente com outros colaboradores do Ubuntu, upstream e outros projetos relacionados ao Xubuntu. Os membros da equipe devem tomar parte na discussão de desenvolvimento fora do Xubuntu, bem como para criar e manter relacionamentos saudáveis com outros projetos e seus desenvolvedores.

Resolução de disputas

O Código de Conduta (CoC) é um dos documentos mais fundamentais na comunidade Ubuntu, incluindo Xubuntu, e funciona como a base para um trabalho comunitário. Todo mundo e todos os meios de comunicação na comunidade Xubuntu são compatíveis com CoC em todos os momentos.

Para ler a revisão mais recente do código de conduta Ubuntu,refira-se a página Código de conduta do Ubuntu no website do Ubuntu

O conselho da comunidade Ubuntu pode ser solicitado a atuar como mediadores/assessores onde seus conhecimentos seriam úteis quando o conselho Xubuntu requer assistência. O conselho Xubuntu e a equipe Xubuntu são encorajados a conversar com outras equipes da comunidade Ubuntu se isso ajudar a resolver o problema.

Quando as disputas ocorrem, os contribuintes são fortemente encorajados a abster-se de tomar as ações contestadas para evitar sabotar o processo de resolução de disputas.

Ciclos de liberação e suporte

O ciclo de lançamento do Xubuntu consiste em 6 meses do início ao lançamento, em seguida, um mês adicional de 9 meses de manutenção para versões padrão e 3 anos de suporte para lançamentos LTS. Embora o próprio projeto Xubuntu não possa oferecer suporte comercial ou garantias comerciais, a equipe Xubuntu fará todos os esforços para estar em conformidade com o período de suporte reconhecido e fornecer atualizações de segurança para pacotes Xubuntu.

Para obter mais informações sobre os detalhes dos processos relacionados ao desenvolvimento de uma versão, consulte Capítulo 6, Processos.

Sementes e composição

Componentes do núcleo

Os pacotes a seguir e suas dependências técnicas são considerados essenciais na composição de um sistema operacional que representa os objetivos definidos anteriormente neste documento:

  • xfwm4 (gerenciador de janelas do Xfce4)

  • xfdesktop4 (área de trabalho do Xfce4)

  • xfce4-panel (painel do Xfce4)

  • xfce4-session (gerenciador de sessão do Xfce4)

  • xfconf (sistema de armazenamento de configuração do Xfce4)

  • xfce4-settings (gerenciador de configurações do Xfce4)

  • thunar (gerenciador de arquivos)

A menos que os componentes mencionados acima estejam depreciados e deixem de funcionar como pretendem, não devem ser substituídos ou removidos da semente.

O meta-pacote xubuntu-core consiste nos pacotes principais, bem como em outros pacotes importantes que contribuem para a experiência do Xubuntu. O meta-pacote xubuntu-desktopdepende do meta-pacote principal e inclui pacotes adicionais para fornecer a experiência completa, pronta a usar,do Xubuntu.

Seleção de pacotes

Ao considerar a seleção de pacotes para Xubuntu, existem duas diretrizes principais:

  • O pacote deve estar em linha com a declaração de missão

  • O processo de seleção de pacotes deve ser conduzido o mais objetivamente possível

Enquanto a linha técnica Xubuntu tem a última palavra sobre a seleção de pacotes com a aprovação do conselho Xubuntu, recomenda-se que toda a equipe esteja envolvida na discussão, trazendo argumentos bem pensados tanto para e contra. Quando apropriado, os testes devem ser conduzidos para coletar medições significativas.

Antes de semear uma aplicação em Xubuntu, os seguintes aspectos devem ser considerados:

  • Usabilidade e integração

  • Utilidade

  • Consumo e tamanho dos recursos

  • Confiança

  • Localização

Ao medir se um pacote é certo para Xubuntu, a decisão não deve ser baseada em uma análise do pacote sozinho, mas o pacote de destino, juntamente com suas dependências. Qualquer embalagem que seja considerada para semeadura deve satisfazer os seguintes requisitos:

  • O pacote deve usar o kit de ferramentas GTK

  • O pacote não deve puxar em bibliotecas pesadas, especialmente se eles vão executar e/ou iniciar frequentemente

  • O pacote deve ser mantido pelo upstream

As diretrizes acima podem ser omitidas com a aprovação do conselho quando há amplos benefícios para semear o pacote.
